# CoExistree 用户指南

## 我们解决什么问题？

### 企业知识管理的常见痛点

| 痛点 | 真实场景 | CoExistree 解决方案 |
|------|---------|---------------------|
| **文档散落在各处** | 产品文档在 Confluence，技术文档在 GitLab，需求在飞书，新人入职要开 5 个账号才能看全 | 统一的知识树模型，所有文档按系统聚合，一处入口全览全貌 |
| **权限管理混乱** | 核心架构文档全员可见，离职员工仍能访问历史文档，敏感信息泄露风险高 | 细粒度权限控制：系统级角色 + 文档级安全等级，最小权限原则 |
| **知识传承断层** | 核心员工离职，交接文档零散不全，继任者翻遍聊天记录才能还原设计思路 | 知识树跟随系统演进自动归档，变更历史可追溯，知识不随人走 |
| **文档与系统脱节** | 代码改了 3 个月，文档还是 1.0 版本；新人按文档操作发现接口早变了 | 文档即变更记录，每次更新都是系统成长快照，保证知识时效性 |
| **检索效率低下** | 在几百份文档中找答案，打开 10 个标签页才能确认正确信息 | AI 智能问答，基于知识树精准定位，答案附带出处，有据可查 |

## 为什么选择 CoExistree？

### 🔐 企业级的安全管控

**五级安全等级体系**

```
公开(1) → 内部(2) → 机密(3) → 高度机密(4) → 绝密(5)
```

- 文档上传时指定安全等级
- 用户查看等级与文档等级匹配才能访问
- MAINTAINER 只能创建不高于自身等级的文档
- 从根源杜绝越权访问敏感信息

**三级角色权限模型**

| 角色 | 权限范围 | 适用场景 |
|------|---------|---------|
| **OWNER** | 系统最高权限，可管理所有成员 | 系统负责人、架构师 |
| **MAINTAINER** | 可上传文档、管理 SUBSCRIBER | 核心开发人员 |
| **SUBSCRIBER** | 只读访问，受限查看等级 | 外包人员、实习生、跨团队协作 |

权限层层下放：
- OWNER 可添加 MAINTAINER 和 SUBSCRIBER
- MAINTAINER 只能添加 SUBSCRIBER
- 避免管理员疲于应对成员申请

### 🌲 知识随系统共同成长

传统文档管理的问题：文档是静态资料，写完就忘，越积越多变成"文档坟墓"。

CoExistree 的做法：
- 每个系统拥有独立的知识树，文档按结构自动组织
- 每次文档更新都是系统的"成长记录"
- 自动维护版本快照，随时回溯历史
- 知识随系统演进自然沉淀，不再丢失

### 💬 可信的智能问答

区别于通用 AI 助手：

| 通用 AI | CoExistree 智能问答 |
|---------|---------------------|
| 可能编造信息 | 只在已上传文档中检索，不编造 |
| 答案无从考证 | 每个回答标注引用来源，点击查看原文 |
| 不了解你的业务 | 基于你的知识库，理解业务语境 |
| 全员看到同样内容 | 权限感知，用户只能看到授权范围内的内容 |

### 👥 优雅的团队协作

- **成员自助管理**：OWNER 和 MAINTAINER 可自行添加团队成员，无需平台管理员介入
- **跨系统协作**：一个用户可参与多个系统，每个系统独立权限
- **安全交接**：人员变动时，系统知识树完整保留，不影响团队运转

## 适用场景

### 🏢 企业研发团队

- 微服务架构下几十个子系统的知识管理
- 技术文档、API 文档、架构设计的统一存储与检索
- 外包人员安全接入，只可见必要信息

### 🎓 技术中台/平台团队

- 平台能力文档的版本管理
- 各业务线接入团队的权限隔离
- 新成员快速了解平台能力

### 🚀 初创公司

- 从第一天开始积累可传承的知识资产
- 创始人离职后，产品知识不会断层
- 随着团队扩张灵活控制文档可见范围

## 核心功能

### 系统管理
- 创建系统，生成独立知识空间
- 成员管理：添加/移除/修改角色
- 查看等级设定：控制成员可见文档范围

### 文档中心
- Markdown 文档导入
- 上传时标记安全等级
- 版本历史查看
- 权限过滤：只显示用户可见的文档

### 知识树
- 自动解析文档结构，构建树形知识图谱
- 可视化浏览与导航
- 变更合并：智能识别文档变更并合并
- 快照回溯：查看任意历史版本

### 智能问答
- 上下文对话：支持多轮交互
- 引用溯源：答案附带原文出处
- 权限感知：只基于用户可见内容回答
- 精准定位：基于知识树节点检索

### 用户管理（管理员）
- 创建/删除用户
- 密码重置
- 角色分配
